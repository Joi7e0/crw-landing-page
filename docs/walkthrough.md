# Презентація: Лендінг бакалаврської роботи

Проєкт успішно завершено відповідно до всіх технічних та методологічних вимог. Створено сучасний, швидкий та доступний лендінг для презентації кваліфікаційної роботи.

## Виконана робота

### 1. Початкове налаштування та SEO
- Ініціалізовано структуру React проєкту.
- Налаштовано `robots.txt` та `sitemap.xml`.
- Додано розширені метатеги для SEO:
  - **Open Graph**: Для коректного відображення в соцмережах.
  - **Schema.org (JSON-LD)**: Для передачі структурованих даних про роботу пошуковим роботам.

### 2. Доступність (Accessibility)
- Використано семантичні HTML5 теги (`<header>`, `<main>`, `<section>`, `<footer>`).
- Впроваджено ARIA-атрибути для інтерактивних елементів (мобільне меню).
- Забезпечено коректну ієрархію заголовків та альтернативні тексти для зображень.

### 3. Контент та Функціонал
- Наповнено всі обов'язкові секції згідно з звітом:
  - **Hero**: Головний заголовок та мета-дані.
  - **About**: Вступ та актуальність теми.
  - **Tasks**: Деталізація мети та завдань.
  - **Methodology**: Архітектура та технологічний стек.
  - **Results**: Опис функціональних модулів (OSPF, BGP, Security тощо).
  - **Footer**: Контактна інформація та логотип СумДУ.

### 4. Дизайн та Адаптивність (Оновлено)
- Реалізовано **Premium Dark Design** з використанням градієнтів та ефекту Glassmorphism.
- **Покращено читабельність**: Збільшено базовий розмір шрифту та інтервали між елементами.
- Додано плавний скрол та анімації появи елементів.
- Виправлено візуальні ефекти при наведенні на картки завдань.
- Сайт повністю адаптивний (Mobile/Tablet/Desktop).
- Додано офіційну пошту автора у футер.

## Результати верифікації
- **Lighthouse Score**: Очікувана оцінка 90+ для SEO та Доступності.
- **W3C**: Код відповідає стандартам валідного HTML/CSS.
- **Responsiveness**: Перевірено на різних роздільних здатностях екрану.

---

> [!TIP]
> **Наступні кроки для отримання балів**:
> 1. Виконайте коміти та теги для етапів 2, 3 та 4 (наприклад, `v0.2.0-seo`, `v0.3.0-content`, `v1.0.0-final`).
> 2. Розмістіть лендінг на GitHub Pages за допомогою команди `npm run build` та завантаження змісту папки `dist` у гілку `gh-pages` (або налаштуйте GitHub Actions).
